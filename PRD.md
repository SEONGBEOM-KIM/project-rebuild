# **8. 🎨 Phaser 설계**

## **✔️ 기본 화면 구성**

- Phaser3 기반으로 전체 게임 화면을 구성한다.
- 해상도는 **1920×1080 기준**으로 설계한다.
- 지도 화면은 Phaser의 카메라 시스템을 활용하여 확대·축소 및 드래그 이동이 가능하도록 구성한다.
- 스토리, 대화, 정책 선택, 결과 피드백도 Phaser Scene 안에서 처리하여 하나의 게임 흐름처럼 이어지도록 한다.

---

## **✔️ 타일 기반 지도 구조**

- 지역 지도는 타일 기반으로 구성한다.
- 기본 타일 크기는 **32px**로 설정한다.
- 실제 배치 가능 영역은 약 **30×18 ~ 40×20 타일** 범위로 구성한다.
- 각 타일은 다음 정보를 가진다.

```jsx
{
  type: "residential" | "commercial" | "industrial" | "forest" | "river" | "road" | "empty",
  buildable: true,
  occupied: false,
  zone: "center" | "outskirts" | "nature" | "traffic"
}
```

---

## **✔️ 건물 배치 규격**

- 건물은 하나의 타일이 아니라 여러 타일을 차지하는 footprint 방식으로 배치한다.

| 구분      | 크기 | 예시                     |
| --------- | ---- | ------------------------ |
| 소형 시설 | 2×2  | 버스정류장, 작은 공원    |
| 중형 시설 | 2×3  | 청년센터, 재활용센터     |
| 대형 시설 | 3×3  | 공장, 산업단지, 관광시설 |

- 건물 배치 시 footprint 전체가 설치 가능 영역인지 확인한다.
- 하나라도 설치 불가 타일이 포함되면 배치할 수 없다.

---

## **✔️ 렌더 레이어 구조**

Phaser Scene 안에서 다음 순서로 레이어를 구성한다.

1. **Background Layer**
   - 지역 배경 지도
   - 산, 강, 도로, 주거지 등 기본 지형 표현
2. **Tile/Grid Layer**
   - 배치 모드에서만 표시
   - 설치 가능 영역은 초록색, 불가능 영역은 빨간색으로 표시
3. **Building Layer**
   - 이미 설치된 시설 표시
   - 설치된 시설은 이후 에피소드에서도 유지
4. **Preview Layer**
   - 선택한 시설의 반투명 이미지 표시
   - 마우스 위치에 따라 타일 단위로 이동
   - 설치 가능 여부에 따라 초록/빨강 강조 효과 적용
5. **Effect Layer**
   - 시설 설치 시 먼지, 빛, 확대/축소, 반짝임 등 이펙트 출력
   - 지역 변화 효과 표현
6. **UI Layer**
   - 대화창, 상태바, 정책 카드, 결과창 등 게임 UI 표시

---

## **✔️ 카메라 조작**

- 마우스 휠을 사용해 지도를 확대·축소한다.
- 마우스 드래그로 지도 화면을 상하좌우 이동한다.
- 최소/최대 확대 비율을 설정하여 지나치게 확대되거나 축소되지 않도록 한다.

예시:

```jsx
minZoom: 1,
maxZoom: 2.5
```

- 카메라 이동 범위는 지도 경계 안으로 제한한다.

---

## **✔️ 배치 모드 인터랙션**

시설 선택 후 배치 모드로 전환되면 다음 기능이 활성화된다.

1. 지도 위에 타일 격자가 표시된다.
2. 마우스 위치에 선택한 시설의 preview 이미지가 나타난다.
3. preview는 가장 가까운 타일 좌표에 자동으로 맞춰진다.
4. 설치 가능 위치면 초록색으로 표시된다.
5. 설치 불가능 위치면 빨간색으로 표시된다.
6. 설치 가능 위치를 클릭하면 시설이 배치된다.
7. 배치 후 상태값과 지역 변화가 반영된다.

---

## **✔️ 설치 가능/불가 판정**

시설 배치 가능 여부는 다음 조건으로 판단한다.

- 배치하려는 footprint 전체가 지도 범위 안에 있는가
- 해당 타일들이 비어 있는가
- 해당 시설이 허용된 구역인가
- 강, 산, 도로 등 설치 불가 지형과 겹치지 않는가
- 특정 시설의 경우 필요한 조건을 만족하는가

예시:

| 시설       | 설치 조건                            |
| ---------- | ------------------------------------ |
| 공장       | 산업 구역 또는 외곽 지역에 설치 가능 |
| 공원       | 주거지역 근처 설치 가능              |
| 청년센터   | 중심지 또는 주거지역 근처 설치 가능  |
| 재활용센터 | 주거지역과 너무 가까우면 불가        |
| 버스정류장 | 도로 인접 타일에 설치 가능           |

---

## **✔️ 상태값 반영**

시설 설치 후 다음 상태값에 영향을 준다.

```jsx
{
  population: +50,
  economy: +20,
  environment: -10,
  satisfaction: +5,
  budget: -100,
  traffic: +10
}
```

- 시설 종류와 설치 위치에 따라 효과가 달라진다.
- 예를 들어 공장을 도로 가까이에 설치하면 경제 효과가 증가하고, 주거지역 가까이에 설치하면 만족도가 감소한다.

---

## **✔️ 장면 전환**

- Phaser Scene을 활용하여 스토리, 선택, 배치, 결과 장면을 전환한다.
- 장면 전환 시 fade in/out, camera pan, zoom 효과를 사용하여 하나의 게임처럼 이어지는 느낌을 준다.

예시 Scene:

```jsx
BootScene;
TitleScene;
StoryScene;
MapScene;
PlacementScene;
ResultScene;
EndingScene;
```

---

## **✔️ 저장 데이터**

배치 결과는 이후 에피소드와 평가에 활용할 수 있도록 저장한다.

```jsx
{
  episode: 3,
  buildingId: "factory_01",
  buildingType: "factory",
  position: {
    x: 12,
    y: 8
  },
  footprint: {
    width: 3,
    height: 3
  },
  effects: {
    population: 120,
    economy: 40,
    environment: -20,
    satisfaction: -5
  }
}
```

## **14. 🔧 추가 시스템 설계 및 확장 요소**

### **✔️ 시간 흐름 시스템**

- 게임은 턴 기반 시간 흐름 구조로 진행된다.
- 각 에피소드 또는 정책 실행 후 일정 시간이 경과한다.
- 일부 정책 효과는 즉시 반영되며, 일부 효과는 시간이 지나며 점차 반영된다.
- 시간 흐름에 따라 인구 변화, 환경 변화, 주민 반응 등이 달라질 수 있다.

예시:

```jsx
{
  currentYear: 2035,
  turn: 3
}
```

---

### **✔️ 상태 UI 시스템**

- 게임 화면 상단 또는 좌측에 지역 상태 정보를 표시한다.
- 학생은 자신의 선택에 따라 상태값이 변화하는 모습을 실시간으로 확인할 수 있다.

표시 항목 예시:

- 인구
- 경제 수준
- 환경 상태
- 주민 만족도
- 예산
- 교통 혼잡도
- 오염도
- 상태값 변화 시 시각적 효과를 제공한다.

예시:

- 증가 → 초록색 강조 애니메이션
- 감소 → 빨간색 강조 애니메이션

---

### **✔️ 연속 배치 및 영향 시스템**

- 이전 에피소드에서 설치한 시설은 이후 에피소드에서도 유지된다.
- 이미 설치된 시설은 이후 발생하는 문제와 결과에 영향을 준다.
- 동일한 정책이라도 기존 지역 상태와 시설 배치에 따라 결과가 달라질 수 있다.

예시:

- 공장 밀집 지역 → 오염 증가 속도 상승
- 공원 부족 지역 → 주민 만족도 감소
- 도로 부족 상태 → 교통 혼잡 발생 가능성 증가

---

### **✔️ 선택의 연쇄 반응 구조**

- 하나의 정책 선택은 긍정적 효과와 부정적 효과를 동시에 발생시킬 수 있다.
- 학생은 단순한 성장보다 균형 있는 발전을 고려해야 한다.

예시:

- 공장 유치 → 경제 성장 증가 / 환경 악화
- 도로 확장 → 교통 완화 / 자연 공간 감소
- 공원 조성 → 만족도 증가 / 예산 감소

---

### **✔️ 엔딩 분기 시스템**

학생의 정책 선택과 지역 상태에 따라 다양한 엔딩으로 분기된다.

### 성공 엔딩

- 인구 증가
- 경제 성장
- 환경 유지
- 주민 만족도 안정

### 과잉 성장 엔딩

- 경제는 크게 성장
- 환경 및 삶의 질 악화
- 주민 불만 증가

### 침체 엔딩

- 환경은 유지
- 경제 성장 실패
- 인구 감소 지속

### 균형 성장 엔딩

- 경제, 환경, 만족도의 균형 유지
- 지속 가능한 발전 달성

---

### **✔️ 시각 변화 시스템**

- 정책 및 시설 배치 결과에 따라 지역의 모습이 변화한다.
- 학생이 선택의 결과를 직관적으로 확인할 수 있도록 구성한다.

예시:

- 인구 증가 → 건물 활성화, 거리 인구 증가
- 경제 성장 → 상권 활성화, 차량 증가
- 환경 악화 → 연기, 회색 톤 증가
- 환경 회복 → 녹지 확대, 하천 색 변화

### **✔️ 몰입형 장면 전환 시스템**

- Phaser Scene 기반으로 모든 장면을 자연스럽게 연결한다.
- 화면 전환 시 fade, zoom, camera 이동 효과 등을 활용하여 하나의 게임처럼 이어지는 경험을 제공한다.

예시:

- StoryScene → PlacementScene 카메라 이동
- 정책 선택 후 결과 화면 fade 전환
- 문제 발생 시 shake effect 적용
