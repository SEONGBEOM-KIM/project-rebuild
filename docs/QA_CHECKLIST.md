# Project Rebuild Manual QA Checklist

자동 검증은 `npm run verify`가 담당하고, Phaser 화면 전환/브라우저 기능은 아래 수동 기준으로 확인합니다.

## 사전 조건

루트 디렉터리에서 실행합니다.

```bash
npm run verify
npm run dev
```

브라우저 접속:

```text
http://127.0.0.1:5173/
```

성공 기준:

- `npm run verify`에서 `Smoke tests passed` 출력
- production build 성공
- 브라우저 개발자 도구 Console에 새 runtime error 없음

## 1. 기본 학습 흐름

| 단계 | 조작 | 성공 기준 |
| --- | --- | --- |
| 제목 | `시작하기` 클릭 | 로그인/회원가입 UI 화면으로 이동 |
| 인증 UI | 진행 버튼 클릭 | 스토리 화면으로 이동 |
| 스토리 | `지역 탐색 시작` 클릭 | 지역 탐색 화면으로 이동 |
| 탐색 | 장소 3곳 이상 클릭 | 버튼이 `자료 확인`으로 바뀜 |
| 탐색 | `자료 확인` 클릭 | 자료 확인 화면으로 이동 |
| 자료 | `원인 질문 풀기` 클릭 | 원인 퀴즈 화면으로 이동 |
| 원인 퀴즈 | 선택지 1개 클릭 | 피드백 표시, 버튼이 `문제 정리`로 바뀜 |
| 원인 퀴즈 | `문제 정리` 클릭 | 문제 정리 화면으로 이동 |
| 문제 정리 | `회복 방향 선택` 클릭 | 회복 방향 선택 화면으로 이동 |
| 방향 선택 | 정책 카드 선택 후 `배치 연습 시작` 클릭 | 배치 화면으로 이동 |
| 배치 | 건물 3개 배치 | `종합 결과 확인` 가능 |
| 결과 | `부작용 확인` 클릭 | 부작용/주의 신호 화면으로 이동 |
| 부작용 | `생각 정리` 클릭 | 생각 정리 화면으로 이동 |
| 생각 정리 | 선택지 1개 클릭 후 완료 | 마무리 화면으로 이동 |

## 2. 배치 시스템

| 항목 | 성공 기준 |
| --- | --- |
| 건물 카드 | 청년센터, 버스정류장, 작은 공원 3종 표시 |
| 추천 배지 | 선택한 정책의 추천 건물에 `추천` 표시 |
| 프리뷰 | 배치 가능 위치는 초록, 불가능 위치는 빨강 표시 |
| 도로 조건 | 버스정류장은 도로 인접 위치에서만 배치 가능 |
| 자연 조건 | 작은 공원은 숲 또는 강 인접 위치에서만 배치 가능 |
| 점유 조건 | 이미 배치된 footprint 위에는 다시 배치 불가 |
| 상태값 | 배치 후 인구/경제/환경/만족도/예산/교통/오염 변화 표시 |
| 효과 마커 | 배치된 위치에 효과 마커 표시 |
| 결과 이동 | 3개 배치 후 결과 화면으로 이동 가능 |

## 3. 결과/마무리 화면

| 화면 | 성공 기준 |
| --- | --- |
| 결과 | 종합 평가, 선택 경향, 정책 방향 일치도 표시 |
| 부작용 | 현재 상태 기준 주의 신호 또는 문제 없음 안내 표시 |
| 생각 정리 | 보완 우선순위 선택 전에는 완료 진행 불가 |
| 마무리 | 선택 요약, 상태 요약, 학습 기록 표시 |
| 처음부터 다시 | `BootScene -> TitleScene`으로 복귀 |
| 배치 다시 조정 | 배치 화면으로 복귀 |

## 4. 학습 데이터/저장

| 항목 | 성공 기준 |
| --- | --- |
| 학습 데이터 보기 | JSON 형태 학습 기록 표시 |
| 임시 저장 | localStorage 저장 완료 메시지 표시 |
| JSON 복사 | 클립보드 권한 허용 시 복사 성공 메시지 표시 |
| JSON 다운로드 | `project-rebuild-ep1-learning-data.json` 다운로드 시도 |
| 저장 데이터 확인 | 제목 화면에서 저장 데이터가 있으면 확인 버튼 표시 |
| 이어보기 | 저장 데이터에서 마무리 화면으로 복원 |
| 저장 삭제 | 저장 데이터 삭제 후 제목 화면에서 저장 확인 버튼 사라짐 |
| JSON 가져오기 | 유효한 학습 JSON을 가져오면 저장 데이터 화면으로 이동 |
| 잘못된 JSON | 오류 메시지를 표시하고 화면이 멈추지 않음 |

## 5. API/mock 제출

| 항목 | 성공 기준 |
| --- | --- |
| API 미리보기 | snake_case payload 표시 |
| API 계약 | `/api/learning-records/` 요청/응답 예시 표시 |
| Mock 제출 | 성공 응답과 제출 ID 표시 |
| 제출 로그 | 최신 제출 기록이 상단에 표시 |
| 로그 삭제 | Mock 제출 로그가 비워짐 |
| 저장/로그 관리 | 저장 데이터와 Mock 제출 로그를 한 화면에서 관리 가능 |

## 6. 교사용 요약

| 항목 | 성공 기준 |
| --- | --- |
| 교사용 요약 | 탐색 장소, 퀴즈, 정책, 배치, 최종 상태 표시 |
| TXT 복사 | 클립보드 권한 허용 시 복사 성공 메시지 표시 |

| 돌아가기 | 마무리 또는 학습 데이터 화면으로 이동 가능 |

## 7. 실패로 판단하는 기준

다음 중 하나라도 발생하면 실패로 봅니다.

- 버튼 클릭 후 아무 반응 없음
- 마우스 포인터가 바뀌었는데 클릭 이벤트가 실행되지 않음
- Console에 `TypeError`, `ReferenceError`, Scene 생성 오류 발생
- 화면 전환 후 검은 화면 또는 멈춤
- 3개 배치 후 결과 화면으로 이동 불가
- 저장/복원 후 필수 학습 기록이 사라짐
- API payload 화면 또는 mock 제출 화면에서 runtime error 발생

## 8. 권장 확인 순서

빠른 회귀 확인:

```text
시작하기 → 스토리 → 탐색 3곳 → 자료 → 퀴즈 → 문제 정리 → 방향 선택 → 건물 3개 배치 → 결과 → 부작용 → 생각 정리 → 마무리
```

저장/API 확인:

```text
마무리 → 학습 데이터 보기 → 임시 저장 → API 미리보기 → Mock 제출 → 제출 로그 → API 계약 → 마무리
```

복원 확인:

```text
처음부터 다시 → 제목 → 저장 데이터 확인 → 이어보기
```
